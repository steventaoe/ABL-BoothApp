# 5 分钟极速上手

软件的一个核心理念是：**不依赖场地网络**，只用一个热点就能让多台设备在局域网里协同工作。  
下面按“现场真实流程”带你从 0 搭好一个可用的出摊系统。

:::tip 我们的架构总共有三种角色
- **管理端**负责：展会前后使用，商品和库存管理、订单审阅、导出报表  
- **顾客端**负责：摆在摊位前面，展示菜单、自助下单/扫码支付  
- **摊主端**负责：拿在摊主手里，接单提醒、确认、配货提示
:::

---

## 你需要准备什么

### 最小可运行配置（推荐）
- 设备 A：安装了**摊盒APP**的平板，同时作为管理端（展前后）和顾客端（展会中）
- 设备 B：有浏览器的手机，扫码连接，作为摊主端接单使用

### 更舒适的展位配置（高人流配置）
- 主机端：电脑（放桌下 / 摊位后方）
- 顾客端：平板（摆摊位前做电子菜单）
- 摊主端：1～3 台手机（摊主/社员同时接单）

## Step 1：软件安装与基础配置

<div style="text-align: center;">
  <img src="../images/help/p1.png" alt="Step 1" style="max-width: 200px;">
</div>
1. 从我们的官网Release页面下载并安装/解压 摊盒APP （记得区分 Windows / Android 版本）
2. 打开应用，进入 **管理员页面（最高控制台）**

:::info 第一次打开建议先做两件事
- 修改管理员密码（默认“admin123”），防止他人误操作
- 点击“获取局域网二维码”按钮，方便后续连接
:::

---

## Step 2：建立制品信息

从侧边栏进入 **全局商品库**：

- 新建商品：编号 / 名称 / 单价 / 分类（可选）/ 图片（可选）



:::tip 推荐的商品录入策略
“全局商品库”提供的是你所有商品的**可复用信息模板**，你最好找一天集中把社团的所有商品都录入进去。  
展会前只需要在“展会-商品管理”里选择实际售卖的种类和数量即可
:::

---

## Step 3: 创建展会，录入摊位商品

<div style="text-align: center;">
  <img src="../images/help/p2.png" alt="Step 3" style="max-width: 200px;">
</div>

从侧边栏回到 **展会管理**：

1. 新建展会：名称 + 日期 + 地点等备注
2. 给展会上传 **收款码图片**，顾客点单后会跳出来它以进行支付
3. 点击展会卡片，进入 **商品管理**：
   - 从“全局商品库”选择本次展会要卖的商品
   - 设置每个商品的初始库存数量和摊位售价（可不同于全局单价）
4. 回到展会首页，启动展会（点击“开始展会”按钮），此时它可以被顾客和摊主页面找到

## Step 4：组网（离线局域网）

<div style="text-align: center;">
  <img src="../images/help/p3.png" alt="Step 4" style="max-width: 200px;">
</div>

我们的多机同步依赖局域网，最稳的方式是 **开个人热点**：

1. 用任意手机打开热点（不需要开流量）
2. **所有设备连接同一个热点 Wi-Fi**
3. 主机端保持应用打开（作为局域网内的“数据源”）
4. 手机通过浏览器，扫描刚刚平板上面显示的**局域网二维码**
5. 浏览器访问可以操作与应用几乎全部相同的界面和功能，这样更加灵活

:::warning 常见坑：不要让设备连到场馆 Wi-Fi 或 校园网
- 场馆 Wi-Fi 通常拥堵且断连频繁，反而会影响稳定性。
- 校园网等大型网络通常启用了客户端隔离，导致设备间无法互联。
:::

## Step 5：配置APP页面显示

<div style="text-align: center;">
  <img src="../images/help/p4.png" alt="Step 5" style="max-width: 200px;">
</div>

由于**APP和网页访问**的功能几乎完全相同，实际上哪台设备打开app作为主机端并不重要。  
你可以根据现场需求灵活分配角色：

### 点单平板
用途：摆摊位前展示菜单、大图浏览、自助下单（可选扫码支付）。

操作思路：
1. 在平板打开APP（或者用浏览器扫描其它打开APP设备的二维码）
2. 选择 **点单页面**
3. 如果展会被设置为开启，你可以发现它的按钮
4. 点进去，即可显示一个标准的菜单

### 摊主手机
用途：接单提醒、确认订单、提示配货。

操作思路：
1. 在手机打开APP（或者用浏览器扫描其它打开APP设备的二维码）
2. 选择 **摊主页面**
3. 输入这个展会的摊主密码（展会创建时可设置，默认为“vendor123”）
4. 点进去即可，等待订单到来进行确认和处理

:::tip 典型的现场流程

1. 顾客端选商品 → 提交订单 → 弹出收款码
2. 摊主端收到提醒 → 确认订单 → 配货  
3. 管理端可以查看所有订单状态，修改信息，导出报表  
:::

如果你是被社团主催拉来帮忙记账的摊主，展会结束后直接进行**导出Excel报表**，就能给社团负责人交一份完整和精美的销售数据啦！

## FAQ：现场常见问题

### Q1：一定要联网吗？
不需要。Booth-Kernel 设计目标就是 **离线局域网可用**：热点即网络。

### Q2：能同时连多少台设备？
一般由你的局域网质量（热点设备性能/现场干扰）决定。  
常见配置：1 主机 + 2 平板 + 3 手机都没问题。

### Q3：掉线会怎样？
- 掉线期间设备可能无法同步最新订单/库存
- 恢复连接后会重新同步（建议现场尽量用热点）

### Q4: 如何备份和导出数据？
- 在**全局商品库**页面，可以导出和导入商品数据的压缩包
- 你可以设置完商品信息后，导出压缩包发给合作者，他们导入后就能快速获得相同的商品信息
